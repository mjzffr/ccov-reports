      17             : /* ~*~ Table Background Painting ~*~
      18             : 
      19             :    Mozilla's Table Background painting follows CSS2.1:17.5.1
      20             :    That section does not, however, describe the effect of
      21             :    borders on background image positioning. What we do is:
      22             : 
      23             :      - in separate borders, the borders are passed in so that
      24             :        their width figures in image positioning, even for rows/cols, which
      25             :        don't have visible borders. This is done to allow authors
      26             :        to position row backgrounds by, for example, aligning the
      27             :        top left corner with the top left padding corner of the
      28             :        top left table cell in the row in cases where all cells
      29             :        have consistent border widths. If we didn't honor these
      30             :        invisible borders, there would be no way to align
      31             :        backgrounds with the padding edges, and designs would be
      32             :        lost underneath the border.
      33             : 
      34             :      - in collapsing borders, because the borders collapse, we
      35             :        use the -continuous border- width to synthesize a border
      36             :        style and pass that in instead of using the element's
      37             :        assigned style directly.
      38             : 
      39             :        The continuous border on a given edge of an element is
      40             :        the collapse of all borders guaranteed to be continuous
      41             :        along that edge. Cell borders are ignored (because, for
      42             :        example, setting a thick border on the leftmost cell
      43             :        should not shift the row background over; this way a
      44             :        striped background set on <tr> will line up across rows
      45             :        even if the cells are assigned arbitrary border widths.
      46             : 
      47             :        For example, the continuous border on the top edge of a
      48             :        row group is the collapse of any row group, row, and
      49             :        table borders involved. (The first row group's top would
      50             :        be [table-top + row group top + first row top]. It's bottom
      51             :        would be [row group bottom + last row bottom + next row
      52             :        top + next row group top].)
      53             :        The top edge of a column group likewise includes the
      54             :        table top, row group top, and first row top borders. However,
      55             :        it *also* includes its own top border, since that is guaranteed
      56             :        to be continuous. It does not include column borders because
      57             :        those are not guaranteed to be continuous: there may be two
      58             :        columns with different borders in a single column group.
      59             : 
      60             :        An alternative would be to define the continuous border as
      61             :          [table? + row group + row] for horizontal
      62             :          [table? + col group + col] for vertical
      63             :        This makes it easier to line up backgrounds across elements
      64             :        despite varying border widths, but it does not give much
      65             :        flexibility in aligning /to/ those border widths.
      66             : */
      67             :
