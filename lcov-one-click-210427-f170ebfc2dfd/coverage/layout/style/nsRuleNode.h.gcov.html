     346             : /**
     347             :  * nsRuleNode is a node in a lexicographic tree (the "rule tree")
     348             :  * indexed by style rules (implementations of nsIStyleRule).
     349             :  *
     350             :  * The rule tree is owned by the nsStyleSet and is destroyed when the
     351             :  * presentation of the document goes away. Its entries are reference-
     352             :  * counted, with strong references held by child nodes, style structs
     353             :  * and (for the root), the style set. Rule nodes are not immediately
     354             :  * destroyed when their reference-count drops to zero, but are instead
     355             :  * destroyed during a GC sweep.
     356             :  *
     357             :  * An nsStyleContext, which represents the computed style data for an
     358             :  * element, points to an nsRuleNode.  The path from the root of the rule
     359             :  * tree to the nsStyleContext's mRuleNode gives the list of the rules
     360             :  * matched, from least important in the cascading order to most
     361             :  * important in the cascading order.
     362             :  *
     363             :  * The reason for using a lexicographic tree is that it allows for
     364             :  * sharing of style data, which saves both memory (for storing the
     365             :  * computed style data) and time (for computing them).  This sharing
     366             :  * depends on the computed style data being stored in structs (nsStyle*)
     367             :  * that contain only properties that are inherited by default
     368             :  * ("inherited structs") or structs that contain only properties that
     369             :  * are not inherited by default ("reset structs").  The optimization
     370             :  * depends on the normal case being that style rules specify relatively
     371             :  * few properties and even that elements generally have relatively few
     372             :  * properties specified.  This allows sharing in the following ways:
     373             :  *   1. [mainly reset structs] When a style data struct will contain the
     374             :  *      same computed value for any elements that match the same set of
     375             :  *      rules (common for reset structs), it can be stored on the
     376             :  *      nsRuleNode instead of on the nsStyleContext.
     377             :  *   2. [only? reset structs] When (1) occurs, and an nsRuleNode doesn't
     378             :  *      have any rules that change the values in the struct, the
     379             :  *      nsRuleNode can share that struct with its parent nsRuleNode.

     382             :  *      when all the values specified are 'inherit'), the nsStyleContext
     383             :  *      can use the same nsStyle* struct as its parent nsStyleContext.
     384             :  *
