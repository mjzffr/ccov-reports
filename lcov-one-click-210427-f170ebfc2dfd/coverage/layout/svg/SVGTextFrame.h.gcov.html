     142             : /**
     143             :  * Frame class for SVG <text> elements.
     144             :  *
     145             :  * An SVGTextFrame manages SVG text layout, painting and interaction for
     146             :  * all descendent text content elements.  The frame tree will look like this:
     147             :  *
     148             :  *   SVGTextFrame                     -- for <text>
     149             :  *     <anonymous block frame>
     150             :  *       ns{Block,Inline,Text}Frames  -- for text nodes, <tspan>s, <a>s, etc.
     151             :  *
     152             :  * SVG text layout is done by:
     153             :  *
     154             :  *   1. Reflowing the anonymous block frame.
     155             :  *   2. Inspecting the (app unit) positions of the glyph for each character in
     156             :  *      the nsTextFrames underneath the anonymous block frame.
     157             :  *   3. Determining the (user unit) positions for each character in the <text>
     158             :  *      using the x/y/dx/dy/rotate attributes on all the text content elements,
     159             :  *      and using the step 2 results to fill in any gaps.
     160             :  *   4. Applying any other SVG specific text layout (anchoring and text paths)
     161             :  *      to the positions computed in step 3.
     162             :  *
     163             :  * Rendering of the text is done by splitting up each nsTextFrame into ranges
     164             :  * that can be contiguously painted.  (For example <text x="10 20">abcd</text>
     165             :  * would have two contiguous ranges: one for the "a" and one for the "bcd".)
     166             :  * Each range is called a "text rendered run", represented by a TextRenderedRun
     167             :  * object.  The TextRenderedRunIterator class performs that splitting and
     168             :  * returns a TextRenderedRun for each bit of text to be painted separately.
     169             :  *
     170             :  * Each rendered run is painted by calling nsTextFrame::PaintText.  If the text
     171             :  * formatting is simple enough (solid fill, no stroking, etc.), PaintText will
     172             :  * itself do the painting.  Otherwise, a DrawPathCallback is passed to
     173             :  * PaintText so that we can fill the text geometry with SVG paint servers.
     174             :  */
