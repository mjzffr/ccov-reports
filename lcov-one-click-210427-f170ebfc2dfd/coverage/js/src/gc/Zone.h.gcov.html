     112             : // A zone is a collection of compartments. Every compartment belongs to exactly
     113             : // one zone. In Firefox, there is roughly one zone per tab along with a system
     114             : // zone for everything else. Zones mainly serve as boundaries for garbage
     115             : // collection. Unlike compartments, they have no special security properties.
     116             : //
     117             : // Every GC thing belongs to exactly one zone. GC things from the same zone but
     118             : // different compartments can share an arena (4k page). GC things from different
     119             : // zones cannot be stored in the same arena. The garbage collector is capable of
     120             : // collecting one zone at a time; it cannot collect at the granularity of
     121             : // compartments.
     122             : //
     123             : // GC things are tied to zones and compartments as follows:
     124             : //
     125             : // - JSObjects belong to a compartment and cannot be shared between
     126             : //   compartments. If an object needs to point to a JSObject in a different
     127             : //   compartment, regardless of zone, it must go through a cross-compartment
     128             : //   wrapper. Each compartment keeps track of its outgoing wrappers in a table.
     129             : //   JSObjects find their compartment via their ObjectGroup.
     130             : //
     131             : // - JSStrings do not belong to any particular compartment, but they do belong
     132             : //   to a zone. Thus, two different compartments in the same zone can point to a
     133             : //   JSString. When a string needs to be wrapped, we copy it if it's in a
     134             : //   different zone and do nothing if it's in the same zone. Thus, transferring
     135             : //   strings within a zone is very efficient.
     136             : //
     137             : // - Shapes and base shapes belong to a zone and are shared between compartments
     138             : //   in that zone where possible. Accessor shapes store getter and setter
     139             : //   JSObjects which belong to a single compartment, so these shapes and all
     140             : //   their descendants can't be shared with other compartments.
     141             : //
     142             : // - Scripts are also compartment-local and cannot be shared. A script points to
     143             : //   its compartment.
     144             : //
     145             : // - ObjectGroup and JitCode objects belong to a compartment and cannot be
     146             : //   shared. There is no mechanism to obtain the compartment from a JitCode
     147             : //   object.
     148             : //
     149             : // A zone remains alive as long as any GC things in the zone are alive. A
     150             : // compartment remains alive as long as any JSObjects, scripts, shapes, or base
     151             : // shapes within it are alive.
     152             : //
     153             : // We always guarantee that a zone has at least one live compartment by refusing
     154             : // to delete the last compartment in a live zone.
